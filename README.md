
# Predictive Asset Maintenance Classifier
> Predicting industrial engine failure before it happens using Random Forest, XGBoost, and a Neural Network — built as part of a Data Science internship application to Shell.

---

## Overview

This project builds and compares three machine learning classifiers to predict whether a turbofan engine will fail within the next 30 cycles, using sensor readings from the **NASA CMAPSS dataset**.

The task mirrors real-world predictive maintenance challenges faced by energy and industrial companies — detecting equipment degradation early enough to schedule maintenance before catastrophic failure occurs.

The project ends with an **interactive Streamlit dashboard** allowing users to explore model performance, per-engine risk scores, SHAP feature importance, and live sensor degradation patterns.

---

## Dashboard Preview

> Run locally with `streamlit run pm_dashboard.py`

The dashboard includes:
- Per-engine failure probability scores (colour coded red/green)
- Model comparison across AUROC and F1-score
- SHAP feature importance — which sensors drive predictions most
- Interactive sensor degradation explorer by engine and sensor

## Methodology

### Dataset
The [NASA CMAPSS](https://www.kaggle.com/datasets/behrad3d/nasa-cmaps) dataset simulates turbofan engine degradation. Each engine runs until failure, with 21 sensor readings recorded at every cycle.

| Subset | Engines | Fault Modes | Conditions |
|---|---|---|---|
| FD001 | 100 train / 100 test | 1 | Sea level |

### Target Variable
**Binary classification** — will this engine fail within 30 cycles?
- `1` = near failure (≤ 30 cycles remaining)
- `0` = healthy (> 30 cycles remaining)

The 30-cycle window gives maintenance teams enough lead time to act.

### Feature Engineering
- Removed 7 low-variance sensors that carry no predictive signal
- Added **5-cycle rolling mean** per sensor to smooth noise and capture degradation trends
- Final feature set: 3 operational settings + 14 sensors + 14 rolling features = **31 features**

### Models

| Model | Key Settings |
|---|---|
| Random Forest | 200 trees, max depth 10, `class_weight='balanced'` |
| XGBoost | 200 estimators, `scale_pos_weight` for imbalance handling |
| Neural Network | 3 hidden layers (128→64→32), ReLU, early stopping |

### Evaluation
- **AUROC** — measures ranking ability regardless of threshold
- **F1-score** — balances precision and recall, critical for imbalanced datasets

Accuracy was deliberately avoided — a model predicting everything as healthy would score ~75% accuracy but miss every failure.

---

## Key Findings

- **XGBoost achieved the best performance** on both AUROC and F1-score — gradient boosting's sequential error correction suits sensor degradation patterns well
- **Rolling mean features were the most important predictors** — raw sensor readings are noisy; smoothed trends carry more signal
- **SHAP analysis** identified the top sensors driving predictions, allowing engineers to focus monitoring on the most informative readings
- The dataset is imbalanced (~75% healthy, ~25% near failure) — `class_weight` and `scale_pos_weight` were essential to prevent models from ignoring the minority class

## How to Run

### 1. Clone the repo
```bash
git clone https://github.com/YOUR_USERNAME/predictive-maintenance.git
cd predictive-maintenance
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Download the dataset
Get the NASA CMAPSS data from [Kaggle](https://www.kaggle.com/datasets/behrad3d/nasa-cmaps) and place the FD001 files in the `data/` folder.

### 4. Run the notebooks in order
```
pm_phase1_data_setup.ipynb  →  generates cleaned data
pm_phase2_eda.ipynb          →  generates featured data + EDA plots
pm_phase3_modelling.ipynb    →  trains models + generates predictions
```

### 5. Launch the dashboard
```bash
streamlit run pm_dashboard.py
# or:
python -m streamlit run pm_dashboard.py
```
